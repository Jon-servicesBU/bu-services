How to Use This Tracker

This tracker lives in your browser — all changes save automatically
Status updates, notes, gate decisions, risk signals, and pipeline values are all saved to your browser storage the moment you change them. You will see a green "Saved" banner in the bottom-right corner confirming each save. No server, no login, no button to click. To share state with your team, email or print the page — full SharePoint List integration is part of the pending P5 task for Eric.

The 6 tabs — what each one is for

1
Dashboard — your home screen
Opens by default. Shows the current sprint actions in the top-left, initiative completion bars top-right, and three columns at the bottom: Blocked (red), In Progress (amber), and Complete (green). Numbers in the column headers update automatically as you change statuses. Use this to start every Commercial Leadership meeting — it answers "what is the team working on right now and what is stuck?"
2
Sprint Board — kanban view of current sprint actions only
Shows the 4 status columns side by side: Not Started, In Progress, Blocked, Complete. Only the actions due in the current sprint period appear here. Click any card to open the detail panel where you can update status and add notes. Use this during weekly check-ins with Vinoth, Max, and Rick Carlton.
3
All Actions — full register of all 38 plan actions
Every action across all 5 initiatives and all sprint periods. Filter by initiative (Mining, Pipeline, Powerline, Nuclear, 3X), by status, or by owner. Use the search box to find a specific account or topic. Change status directly in the dropdown on each row. Add notes in the text field on the right — useful for recording what happened on a call, why something is blocked, or when a follow-up is due.
4
Pipeline — revenue progress against 2026 targets
Top section: drag the slider for each initiative to reflect current pipeline confidence (in $M CAD). The bar turns from grey toward the initiative colour as it fills. Update this weekly before the Commercial Leadership meeting. Bottom section: stacked completion bars showing how many actions per initiative are Not Started / In Progress / Blocked / Complete.
5
Gates — 6 go/no-go decision points
Each gate is a critical decision point from the Execution Attack Plan. Update the status (Pending / Passed / Failed / Partial) when you reach each gate. The card header colour changes with the status — green for Passed, red for Failed, amber for Partial. A Failed gate triggers a contingency plan review at the next Ion/Olivier meeting.
6
Risks — early warning signal monitor
6 risks from the plan (R-01 to R-06) with their early warning signals listed. Click "Mark signal" on any risk to open a checklist and tick which specific signals you are observing. Any triggered signal turns the card border red. Flag every triggered signal at the next Monday Commercial Leadership meeting — do not wait for the next gate.

Updating a status — three ways

A
Dropdown in the All Actions table
Go to All Actions tab. Find the action row. Click the coloured status dropdown and select the new status. It saves instantly. The colour of the dropdown changes to match the status.
B
Click the action title to open the detail panel
In Any Actions or Sprint Board, click the action title text. A modal panel opens showing the full deliverable text, all owners, a status dropdown, and a notes field. Update both status and notes here, then click Save Notes. The modal closes and the table updates immediately.
C
Click a Sprint Board card
In the Sprint Board tab, click any card. The same detail panel opens. This is the fastest route during a meeting — you can update the whole sprint in 2-3 minutes by clicking cards one by one.

What the status colours mean

—
Not Started (grey)
The action has not been picked up yet. Default state for all actions when the tracker is first opened.
▶
In Progress (amber)
Work has started — a call has been made, a proposal is being drafted, a vendor qualification is in flight. Use notes to record where things stand.
✓
Complete (green)
The deliverable has been achieved. For actions with a Salesforce deliverable, mark Complete only after the record is logged in Salesforce — not when the call happens.
✗
Blocked (red)
The action cannot proceed — waiting for a decision, a dependency, an external response, or a resource. Use notes to record exactly what the blocker is. Blocked actions appear in the red column on the Dashboard and must be discussed at Monday's Commercial Leadership meeting.
G
Gate Review (purple)
Used for gate decision actions — the gate is actively under review. Transition to Complete (gate passed) or Blocked (gate failed, contingency activated).

Recommended weekly routine

Monday before the Commercial Leadership meeting (5 minutes)
Open the tracker. Go to Dashboard. Review the Blocked column — anything new since last week? Open All Actions, filter by owner "Vinoth" then "Max" then "Rick" to spot any actions that should have moved but have not. Note any for discussion. Check Risks — any new signals to flag?
Friday afternoon (10 minutes — after Salesforce is updated)
Open All Actions. Filter to "In Progress". For each In Progress action: is it still in progress or is it done? Update accordingly. Add notes on any calls made or proposals issued this week. Update the Pipeline sliders to reflect this week's pipeline movement. The tracker and Salesforce should tell the same story.
At each gate date (before the monthly Gate Review meeting)
Open Gates. For each gate whose date has passed, update the status to Passed, Failed, or Partial. A Failed gate means a contingency plan must be activated — see Section 5 of the Execution Attack Plan document for the relevant pivot plan.

No JSON file needed — data is embedded

This is a single, standalone HTML file
All 38 actions, 6 gates, and 6 risks are embedded directly in this file as JSON data. No separate JSON file, no server, no database. Save this file to SharePoint and open it in any browser — everything works immediately. Your status updates, notes, and pipeline values are stored in the browser on the computer where you make changes. To transfer your progress to another computer, use the Export button (coming in next version) or simply note the key updates.
