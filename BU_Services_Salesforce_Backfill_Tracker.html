Salesforce Backfill Tracker

Every 2026 Opportunity in Salesforce needs the BU_Initiative__c picklist set correctly before Power Automate runs. Without it, initiative revenue shows as $0 in the Command Centre — even if the deal is Closed Won.

Why this matters: Power Automate queries Salesforce using BU_Initiative__c = 'Mining NDT' (etc.) to populate the Command Centre KPIs. Any opportunity without this field set is invisible to the Command Centre — it will not appear in initiative totals, region totals, or the Deck Report. This tracker ensures 100% coverage before go-live.
How to use this tracker
01In Salesforce: run the SOQL query below to export all 2026 Opportunities. Copy the Name, Account, Amount, Stage, and current BU_Initiative__c value for each.
02Add each opportunity to this tracker using the form below. For any opp with BU_Initiative__c already set correctly, mark it Done. For any missing or wrong value, mark the correct initiative and fix it in Salesforce.
03Work through until all opps are tagged. The progress bar turns green when 100% complete. Re-run Power Automate after completing the backfill — the Command Centre will update Sunday night.

SOQL Query

Run in Salesforce Developer Console or Data Loader
SELECT Id, Name, Account.Name, Amount, StageName, CloseDate, BU_Initiative__c, Owner.Name
FROM Opportunity
WHERE CloseDate >= 2026-01-01
  AND CloseDate <= 2026-12-31
  AND IsDeleted = false
ORDER BY CloseDate ASC
