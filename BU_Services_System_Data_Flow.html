Resilience & Fallbacks
If Supabase is unreachable
CRM — saves to localStorage only · Sync dot turns red · "Offline — local only" label · No data lost
Command Centre — falls back to localStorage copy from last successful sync · Stale but not blank
Recovery — next time Supabase is reachable, CRM auto-syncs on next save · CC subscribes and receives immediately
Same-browser fallback — if CRM and CC are in same browser, localStorage storage event still works as secondary channel
If actuals.json is unavailable
Command Centre — loads from CONFIG hardcoded defaults · All manual entries (budget, targets) still show
CRM data — Supabase data unaffected · Pipeline sections still live
Console message — "actuals.json not found — using CONFIG defaults" · No user-visible error
Affected sections — utilisation, headcount, actual revenue, vacation — show CONFIG values until next Sunday sync
Security Model
Supabase — what is exposed
Table: crm_sync only — two rows: pipeline_sync, hub_summary
Key: anon/public key embedded in HTML · Cannot access any other project or table
RLS: Row Level Security enabled · "allow all" policy is safe because no PII or financial detail is stored — only aggregates ($M totals, counts)
What's NOT in Supabase: individual deal names, company names, contact data — those stay in each user's localStorage
actuals.json — access model
Location: SharePoint document library — same location as Power Automate writes
Access: Anyone with SharePoint access can read it · No credentials in HTML files
Content: Aggregated financials only — no individual employee salary data
Future: Graph API auth (phase 2) will restrict to authenticated BU Services team members only
