How to Update the System

Who does what, and when
Jeremy Irwin — Monthly
By 5th of each month · actuals.json on SharePoint
1Open actuals.json on SharePoint (BU Services folder)
2Update nortech.actual with Nortech + RockX invoiced revenue for the month
3Update monthly.{year_month}.nortech for the monthly breakdown chart
4Save file. Command Centre reads it on next page load.
★Annual: Update budget_* fields and eur_rate before January 31.
Jon Eustace — After Each Gate Review
Monthly · actuals.json on SharePoint
1Open actuals.json. Update gates[].status for any milestone that moved (green/amber/grey/red)
2Update risks[].status if any trigger was observed (monitoring → triggered)
3Update nuclear_outages[] if any OPG window date or contract status changed
4Update lastUpdated, updatedBy, version fields. Save.
Commercial Owners — Every Friday by 4PM
Weekly · Salesforce · Power Automate reads Sunday 7PM ET
1All deals closed this week → Stage = Closed Won, Amount in CAD, Close Date = invoice date
2All new prospects → New Opportunity with BU_Initiative__c picklist value
3All active opportunities → Stage and Amount updated
4Power Automate runs Sunday 7PM → actuals.json updated → Command Centre refreshed
Management Hub — When org or process changes
Manual · Edit HTML file · Upload to SharePoint
1Open BU_Services_Management_Hub.html in any text editor
2Org changes: find the org section HTML and update card names/titles/connections
3PEOPLE object in JS: update mandate, initiatives, rhythm, docs for the changed person
4MEETINGS JS var: update chair or attendees if a meeting changes
5DOC_GROUPS: add new files to the library. Save and re-upload to SharePoint.
