What Has Been Built

full system summary

This system is a fully integrated commercial management platform for BU Services (Nucleom + Nortech). It was built in three layers: the strategic documents that define the plan, the Command Centre portal that tracks execution in real time, and the data infrastructure that keeps it current automatically. The three layers are designed to work independently — if any one piece is unavailable, the others continue to function.

The core idea in one sentence
A set of HTML documents anyone can open in a browser, a single JSON file that carries all live data, a Power Automate flow that writes to that file every Sunday, a People & Capacity Dashboard showing overhead %, utilization, and vacation liability for all 126 employees in real time, and a people management system that gives every person a clear lane — so the business runs with structure, visibility, and shared accountability without anyone manually chasing information.

INT BU Bi-Weekly Deck — Automation

what can be auto-populated vs. what stays manual

The INT BU Business Review deck is a 22-slide bi-weekly presentation for the French Executive team. The Command Centre and actuals.json have been extended to carry all the data needed to auto-populate the data-driven slides. A Power Automate deck-generation flow reads actuals.json on the morning of each review and produces a pre-filled PowerPoint — then Jon or Olivier adds narrative commentary on the manual slides before sending.

How deck generation works
A scheduled Power Automate flow runs on the morning of each bi-weekly review. Step 1: it copies all current actuals.json values to the prev_* fields (creating the delta for the variations slide). Step 2: it fetches fresh data from Salesforce and Dynamics. Step 3: it calls a Python script (python-pptx) that reads the template and writes all data fields into the correct slide positions. Step 4: the completed deck lands in the SharePoint BU Services folder, ready for Jon to add commentary. Total generation time: under 2 minutes.

Setting up meeting tracking (2 hours 15 minutes)

1
Part 1: Create the OneNote notebook (30 min)
Go to onenote.com → New notebook → Name: "BU Services — Meeting Tracking". Create seven sections, one per meeting series. Share with leadership team. Then open each recurring Teams meeting, click the Notes tab, and link it to the correct OneNote section. From that point, Teams auto-creates a dated page whenever a meeting starts.
2
Part 2: Create the two SharePoint Lists (45 min)
In SharePoint → New → List. Create BU Services Meeting Log (10 columns including Meeting Series, Date, Status, Action Count) and BU Services Meeting Actions (9 columns including Owner, Due Date, Status). Create three filtered views for Meeting Actions: Open Actions, My Actions, Overdue. Full column specifications are in Section 2 of BU_Services_Meeting_Tracking_Setup.docx.
3
Part 3: Add meeting health to the Sunday flow (60 min)
Open the existing Sunday Power Automate flow. Add a Get Items action reading from the Meeting Actions list filtered to open/in-progress actions. Add seven Compose actions counting actions per meeting series. Add a Compose counting overdue actions. Update the Build JSON action with the eight new actions_* fields. Full expressions are in Section 3 of the Meeting Tracking Setup Guide.
